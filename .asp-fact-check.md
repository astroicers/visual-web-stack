# 外部事實查證紀錄（asp-fact-verify）

> 查證日期：2026-06-11
> 範圍：visual-web-stack skill 所引用的第三方套件 API 與版本

| # | 主張 | 查證結果 | 來源 |
|---|------|---------|------|
| 1 | Anime.js v4 使用具名匯入 `import { animate, createTimeline } from 'animejs'`，全域 `anime` 物件已移除 | ✅ 確認。v4 完全模組化、可 tree-shake，另支援子路徑匯入（`animejs/animation`） | [animejs.com/documentation](https://animejs.com/documentation/timeline/)、[v3→v4 遷移指南](https://github.com/juliangarnier/anime/wiki/Migrating-from-v3-to-v4) |
| 2 | Motion（前 framer-motion）匯入路徑為 `motion/react` | ✅ 確認。套件名 `motion`，`import { motion, AnimatePresence } from 'motion/react'`；framer-motion 仍可用但已停止積極開發 | [motion.dev/docs/react](https://motion.dev/docs/react)、[升級指南](https://motion.dev/docs/react-upgrade-guide) |
| 3 | Lenis 套件名為 `lenis`（非 @studio-freight/lenis），GSAP 整合用 `gsap.ticker.add((t) => lenis.raf(t * 1000))` + `lagSmoothing(0)` + `lenis.on('scroll', ScrollTrigger.update)` | ✅ 確認。官方 README 即此模式；`data-lenis-prevent` 屬性與 `lenis.scrollTo()` API 同步確認 | [github.com/darkroomengineering/lenis](https://github.com/darkroomengineering/lenis)、[npm lenis](https://www.npmjs.com/package/lenis) |
| 4 | React 18 必須配 @react-three/fiber@8；v9 需要 React 19 | ✅ 確認。官方安裝文件明載「fiber@8 pairs with react@18, fiber@9 pairs with react@19」；drei 對應為 v9（drei v10+ 需 fiber v9） | [R3F installation](https://r3f.docs.pmnd.rs/getting-started/installation) |
| 5 | @react-three/postprocessing 2.x 與 fiber v8 相容，提供 SMAA / Bloom / ChromaticAberration / Noise / Vignette | ✅ 確認。2.16.x + fiber 8.17.x 為已驗證組合 | [react-postprocessing docs](https://react-postprocessing.docs.pmnd.rs/)、[npm](https://www.npmjs.com/package/@react-three/postprocessing) |
| 6 | Tailwind CSS 目前 major 為 v4，Vite 整合用 `@tailwindcss/vite` plugin + CSS-first（`@import "tailwindcss"`），不需 tailwind.config.js | ✅ 確認 | [tailwindcss.com/blog/tailwindcss-v4](https://tailwindcss.com/blog/tailwindcss-v4)、[官方 Vite 安裝指南](https://tailwindcss.com/docs) |
| 7 | Vite 目前 major | ⚠️ 修正基準認知：**Vite 8**（2026-03-12 發布，bundler 改為 Rolldown），非 7.x | [vite.dev/blog/announcing-vite8](https://vite.dev/blog/announcing-vite8) |

## 2026-06-11 推版前 review 補充查證

| # | 主張 | 查證結果 | 來源 |
|---|------|---------|------|
| 8 | Lenis 官方建議匯入 `lenis/dist/lenis.css` 而非手抄樣式 | ✅ 確認。官方 stylesheet 含 `[data-lenis-prevent]` 的 `overscroll-behavior: contain`、stopped 狀態、iframe 規則，手抄版容易漏 | [lenis README / packages/core/lenis.css](https://github.com/darkroomengineering/lenis) |
| 9 | @react-three/postprocessing 2.19.x 型別：`ChromaticAberration` 的 `radialModulation`、`modulationOffset` 為必填；`EffectComposer` children 型別 `JSX.Element \| JSX.Element[]` 不接受條件渲染的 `false` | ✅ 以 tsc 實測重現（TS2739 / TS2322），範例已改為陣列組裝＋補必填 props | npm @react-three/postprocessing 2.19.1 d.ts |
| 10 | Radix Dialog 的 Content 必須搭配 `Description`（或顯式 `aria-describedby={undefined}`），否則 console 警告 | ✅ 確認（@radix-ui/react-dialog 1.1.x DescriptionWarning），範例已固定渲染 Description（無視覺描述時 sr-only 代用） | [radix-ui.com Dialog docs](https://www.radix-ui.com/primitives/docs/components/dialog) |
| 11 | React 18 StrictMode 開發模式 effect 順序為 mount → cleanup → mount，cleanup 即足以避免雙 Lenis 實例 | ✅ 確認，相關註解與 pitfalls 條目已修正 | [react.dev/reference/react/StrictMode](https://react.dev/reference/react/StrictMode) |
